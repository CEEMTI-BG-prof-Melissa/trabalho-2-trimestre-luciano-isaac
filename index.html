Abstração

Em Programação Orientada a Objetos (POO), abstração é o processo de ocultar detalhes complexos e irrelevantes, focando apenas nas características e comportamentos essenciais de um objeto para um determinado contexto.

  • Simplificação

  • Foco na Funcionalidade

  • Reutilização de Código

  • Organização

Encapsulamento

O encapsulamento é um conceito fundamental na programação orientada a objetos que permite ocultar os detalhes internos de um objeto e expor apenas o que é necessário para interagir com ele

  • Ocultação de detalhes

  • Flexibilidade e manutenção

  • Reutilização

  • Segurança

Herança

A herança é um conceito fundamental na programação orientada a objetos que permite que uma classe, chamada classe derivada, herde atributos e métodos de outra classe, chamada classe base ou superclasse

  • Reutilização de código

  • Organização do código

  • Extensibilidade

  • Polimorfismo

Polimorfismo

O polimorfismo, um dos pilares da programação orientada a objetos, permite que objetos de diferentes classes sejam tratados como objetos de uma mesma classe base, ou seja, que um mesmo método possa ter comportamentos diferentes dependendo do objeto que o executa.

  • Flexibilidade

  • Reutilização de Código

  • Manutenção

Evento

Em orientação a objetos, um evento é um tipo de mensagem ou notificação que indica que algo aconteceu. Ele pode ser gerado por interações do usuário, como cliques em botões, ou por mudanças no estado de um objeto.

  • A orientação a objetos é um paradigma de programação, enquanto a programação orientada a eventos é um estilo de programação que usa eventos para controlar o fluxo de execução. A combinação dos dois permite construir sistemas mais flexíveis, interativos e com melhor organização de código, especialmente em aplicações que envolvem interação com o usuário ou comunicação entre diferentes componentes.

Classe

Em programação orientada a objetos, uma classe é como um projeto ou modelo que define as características e comportamentos de um tipo de objeto.

  • Uma classe é um modelo que define a estrutura e o comportamento de objetos.

  • Objetos são instâncias de classes.

  • Classes ajudam a organizar e estruturar o código, facilitando a criação e o gerenciamento de objetos complexos.

Estado

Em orientação a objetos, o estado de um objeto refere-se aos seus atributos e seus respectivos valores em um determinado momento.

  • O estado é fundamental para o funcionamento dos objetos e para a lógica de um sistema orientado a objetos.

  • A mudança de estado de um objeto pode desencadear diferentes comportamentos e ações.

  • O estado permite que os objetos se diferenciem e interajam de forma significativa.
